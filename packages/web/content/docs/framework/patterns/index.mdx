## Overview

Patterns are ready-made compositions of Noetic's 7 primitives (`step.run`, `step.llm`, `step.tool`, `branch`, `fork`, `spawn`, `loop`). Each pattern solves a common agent architecture problem without sacrificing the ability to customize.

Every pattern returns a `Step` (usually a `StepLoop`), which means you can nest patterns inside other patterns, wrap them in additional loops, or use them as children in spawn and fork steps.

## Available Patterns

| Pattern | Function | What It Does |
|---|---|---|
| [ReAct](/docs/framework/patterns/react) | `react()` | Observe-Think-Act loop with tools |
| [Ralph Wiggum](/docs/framework/patterns/ralph-wiggum) | `ralphWiggum()` | Self-refining agent with verification |
| [Task Trees](/docs/framework/patterns/task-trees) | `compilePlan()` | Tree-structured multi-agent workflows |
| [Adaptive Plans](/docs/framework/patterns/adaptive-plans) | `adaptivePlan()` | Dynamic planning with error-driven revision |
| [Recursive LLM](/docs/framework/patterns/recursive-llm) | `spawn()` with a fresh child context | Recursive sub-task delegation |
| [Thread Weaving](/docs/framework/patterns/thread-weaving) | `fork(mode: 'all')` | Parallel analysis with merged results |
| [Dual Agent](/docs/framework/patterns/dual-agent) | `spawn()` + `channel()` | Two agents communicating via channels |
| [A2A Remote](/docs/framework/patterns/a2a-remote) | `ExternalChannel` | Inter-process agent communication |

## How Patterns Compose

Patterns are not special. They return plain `Step` objects:

```ts validate=none
import { react, ralphWiggum, fork } from '@noetic-tools/core';

// Use a pattern as a fork path
const parallel = fork({
  id: 'parallel-research',
  mode: 'all',
  paths: () => [
    react({ model: 'openai/gpt-4o', tools: [searchTool] }),
    react({ model: 'openai/gpt-4o', tools: [analyzeTool] }),
  ],
  merge: (results) => results.join('\n\n'),
});

// Wrap a pattern in verification
const verified = ralphWiggum({
  model: 'openai/gpt-4o',
  instructions: 'Write unit tests.',
  tools: [codeTool],
  verify: async (output) => ({
    pass: String(output).includes('expect('),
    feedback: 'Tests must use expect() assertions.',
  }),
});
```

## Choosing a Pattern

- **Single task with tools**: Start with [ReAct](/docs/framework/patterns/react).
- **Quality matters**: Use [Ralph Wiggum](/docs/framework/patterns/ralph-wiggum) to add self-verification.
- **Multi-step plan**: Use [Task Trees](/docs/framework/patterns/task-trees) for static plans or [Adaptive Plans](/docs/framework/patterns/adaptive-plans) for dynamic ones.
- **Parallel work**: Use [Thread Weaving](/docs/framework/patterns/thread-weaving) via `fork`.
- **Sub-tasks**: Use [Recursive LLM](/docs/framework/patterns/recursive-llm) via `spawn`.
- **Agent collaboration**: Use [Dual Agent](/docs/framework/patterns/dual-agent) for in-process or [A2A Remote](/docs/framework/patterns/a2a-remote) for cross-process.
